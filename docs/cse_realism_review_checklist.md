# Exhale CSE Realism Review Checklist

This checklist is the hard authoring standard for Exhale CSE content. The goal is not just clinically correct cases. The goal is cases that feel like the real NBRC CSE: neutral, sequenced, operational, and tightly decision-based.

## How To Use This Checklist

For each case, review:

1. step prompts
2. answer choice construction
3. distractor quality
4. data release cadence
5. outcome text
6. explanation and rationale tone
7. reassessment loop
8. disposition and follow-up logic

Then classify the case as one of:

- `Rewrite In Place`: case structure is sound; wording and options need cleanup
- `Heavy Refactor`: clinical core is usable, but sequencing, options, and scoring need major changes
- `Rebuild`: the decision model is too far from exam style to salvage efficiently

## 1. Core Experience Test

The case should feel like:

- a simulation
- a sequence of clinical decisions
- a test of prioritization and indication logic

The case should not feel like:

- a lecture
- a teaching script
- an over-explained protocol
- a custom educational product with editorial language

## 2. Prompt Tone

Prompt wording should be:

- neutral
- concise
- chart-like
- clinical

Preferred patterns:

- `Which of the following should be evaluated?`
- `Which of the following should be recommended?`
- `Which of the following should be done FIRST?`
- `Which of the following should be recommended postadmission?`

Avoid:

- `best available`
- `structured plan`
- `causes-first response`
- `core surveillance metrics`
- `high-yield exam distinction`
- `bedside-first`

## 3. Sequencing Standard

The case should follow real clinical order:

1. initial presentation
2. immediate stabilization or high-priority assessment
3. targeted evaluation
4. first-line intervention
5. reassessment
6. escalation or de-escalation
7. disposition or follow-up

`FIRST` questions must enforce true immediate priority, not just a generally good option.

## 4. Data Release Standard

Only reveal enough information to support the next decision.

Good pattern:

- initial symptoms and basic vitals
- one decision
- updated findings
- one next decision

Avoid:

- front-loading the whole case
- using metadata reveals that feel game-like instead of chart-like
- showing later-phase meaning too early

## 5. Answer Choice Standard

### Single-choice items

Single-choice options should usually be one action, one test, one treatment, or one management change.

Good:

- `Obtain a chest radiograph.`
- `Begin manual ventilation.`
- `Increase PEEP.`
- `Proceed with endotracheal intubation.`

Bad:

- multi-step bundles
- full management packages
- option text that contains an entire protocol

### Select-many items

These should:

- use smaller realistic limits, usually 2-4
- contain indicated actions plus plausible non-indicated distractors
- force students to separate useful from merely possible

Avoid defaulting to high counts like `MAX 8` unless the real exam structure truly justifies it.

## 6. Distractor Standard

The best distractors are:

- plausible
- tempting
- wrong because of timing, risk, yield, or incompleteness

Distractors should often be:

- a good action at the wrong time
- a useful action that is too incomplete
- a test that is lower-yield than the better next test
- an intervention that delays the correct intervention

Avoid distractors that are obviously absurd unless the contrast is intentionally stark.

## 7. Assessment vs Recommendation Separation

Keep these decision types clean:

- `Assessment / evaluation`: assess, examine, obtain, monitor, trend
- `Recommendation / intervention`: start, recommend, initiate, titrate, escalate

Do not mix both inside one question unless the real clinical step truly requires it.

## 8. Reassessment Standard

After any meaningful intervention, the case should show response and ask the next question from the updated state.

Reassessment can include:

- vitals
- SpO2
- breath sounds
- ABG
- work of breathing
- mental status
- hemodynamics
- ventilator mechanics

If a case lacks a meaningful reassessment loop, it will not feel like the CSE.

## 9. Vocabulary Standard

Use restrained, standard clinical wording:

- `ABG analysis reveals`
- `breath sounds reveal`
- `the patient is difficult to arouse`
- `SpO2 is 88% while receiving an FIO2 of 0.40`

Avoid language that sounds authored for a curriculum instead of a simulation.

## 10. Explanation Standard

Rationales should be short, mechanistic, and scoring-oriented.

Good style:

- `This is indicated to assess...`
- `This treats the likely cause of...`
- `This is not indicated because...`
- `This delays definitive treatment.`

Avoid long tutoring-style rationales.

## 11. Severity And Escalation Standard

Escalation should be tied to specific worsening findings such as:

- falling pH
- rising PaCO2
- worsening hypoxemia
- dropping mental status
- increased work of breathing
- apnea or bradycardia
- worsening hemodynamics

The case should make escalation feel earned by the physiology.

## 12. Monitoring And Testing Yield Standard

Tests and monitoring choices should be included only if they help the next decision.

Ask:

- Does this change management now?
- Is it proportional to severity?
- Is it the highest-yield next test?

## 13. Ventilator Question Standard

Ventilator items should usually ask for:

- one adjustment
- one mode choice
- one initial settings choice
- one response to a new ABG or mechanics change

Avoid packaging a whole ventilator strategy into a normal answer option.

## 14. Disposition And Follow-Up Standard

Disposition and transition questions should remain concrete and operational.

Good follow-up actions:

- monitored admission
- transfer to ICU
- smoking cessation
- inhaler teaching
- pulmonary rehab
- home oxygen plan
- return precautions

Avoid vague management language.

## 15. Scoring Logic Standard

The correct answer should win because it is:

- indicated
- timely
- safe
- high-yield
- proportional to severity

Wrong answers should fail because they are:

- premature
- delayed
- incomplete
- too risky
- not indicated
- lower-yield than the better next action

## 16. Remediation Thresholds

Use this triage when reviewing an existing case.

### Rewrite In Place

Choose this when:

- clinical flow is sound
- step order is realistic
- options are mostly salvageable
- the main problem is tone, wording, or distractor polish

### Heavy Refactor

Choose this when:

- the clinical idea is good
- but prompts are over-authored
- options are too bundled
- select-many counts are too broad
- reassessment needs to be rebuilt
- scoring logic needs significant tightening

### Rebuild

Choose this when:

- the case is built around the wrong decision model
- the sequence does not resemble real CSE logic
- the option structure cannot be repaired cleanly
- fixing it would be messier than starting over

## 17. Batch Rewrite Strategy

Do not rewrite the whole bank blindly. Rewrite by case family:

1. obstructive disease / COPD / asthma
2. ventilator emergencies
3. pulmonary edema / cardiac / PE
4. neuromuscular ventilatory failure
5. trauma
6. neonatal / pediatric

For each batch:

1. audit representative cases
2. define the family-specific language standard
3. refactor prompts and options
4. tighten scoring logic
5. test the updated flow
